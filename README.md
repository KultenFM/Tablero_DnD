# 🐉 Tablero Táctico D&D - Gestor de Encuentros

## 📖 Descripción del Proyecto
Aplicación web interactiva diseñada para optimizar la gestión táctica de monstruos y reducir drásticamente el tiempo de resolución de combates en partidas de Dungeons & Dragons. El sistema automatiza las tiradas de dados y el orden de iniciativa, eliminando la necesidad de usar papel, lápiz y dados físicos por parte del Dungeon Master.

---

## 🎯 1. Objetivos y Metas SMART
* **Objetivo Específico:** Desarrollar una aplicación web interactiva mediante Python y PostgreSQL para gestionar la base de datos de criaturas y el motor de combate.
* **Meta Medible (MVP):** La primera versión estará oficialmente terminada cuando el sistema permita agregar un monstruo a la base de datos y generar automáticamente una tabla de combate táctico de al menos 5 turnos.
* **Problema a Resolver:** Hacer los combates de D&D más fluidos, ágiles y con menos carga cognitiva para el director de juego.

---

## 🚧 2. Alcance del Proyecto (Scope)

**✅ INCLUIDO en esta versión:**
* Base de datos relacional de monstruos con sus estadísticas (Fuerza, Destreza, Ataques, etc.).
* Tablero de combate táctico con orden de iniciativas (Jugadores vs. Monstruos).
* Simulador matemático de tiradas de dados en tiempo real.

**❌ FUERA del alcance (Por ahora):**
* Gestión de inventario de jugadores.
* Mapas interactivos 2D/3D o cuadrículas de movimiento.
* Multijugador online (la app es de uso exclusivo local para el Dungeon Master).
* Integración de manuales completos, lore o videos.

---

## 🛠️ 3. Recursos y Stack Tecnológico
* **Frontend (Interfaz Gráfica):** Streamlit
* **Backend (Lógica y API):** FastAPI
* **Base de Datos:** PostgreSQL
* **Entorno de Desarrollo:** PC Local (Windows) / Visual Studio Code
* **Control de Versiones:** Git y GitHub

---

## ⚠️ 4. Gestión de Riesgos Críticos
* **Riesgo:** Frustración técnica por el uso de herramientas nuevas, errores de configuración de rutas en Python y escasez de tiempo.
* **Plan de Mitigación:** 1. Avanzar paso a paso sin saltar etapas de arquitectura.
  2. Documentar cada avance y cada error superado.
  3. Dedicar ventanas de tiempo estrictas y consistentes: **2 horas, una vez a la semana**.
